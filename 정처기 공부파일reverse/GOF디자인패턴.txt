객체의 인스턴스 생성에 관여하고 클래스 정의와 객체 생성 방식을 구조화, 캡슐화하는 패턴\생성 패턴\1
생성 단계를 캡슐화 해서 구축 공정을 동일하게 이용하도록 하는 패턴\Builder\1
기존 객체를 복제해서 새 객체를 생성할 수 있도록 하는 패턴\Prototype\1
상위 클래스에서 객체를 생성하는 인터페이스를 정의하고 하위 클래스에서 인스턴스를 생성하는 패턴\Factory Method\1
생성군들을 하나에 모아놓고 팩토리 중에서 선택하게 하는 패턴\Abstract Factory\1
유일한 하나의 인스턴스를 보장하도록 하는 패턴\Singgleton\1
클래스나 객체를 조합해 더 큰 구조를 만드는 패턴\구조패턴\1
추상과 구현을 분리해서 결합도를 낮춘 패턴\Bridge\1
소스를 변경하지 않고 기능을 확장하는 패턴\Decorator\1
하나의 인터페이스를 통해 느슨한 결합을 제공하는 패턴\Facade\1
대량의 작은 객체들을 공유하는 패턴\Flyweight\1
대리인이 대신 그 일을 처리하는 패턴\Proxy\1
개별 객체와 복합 객체를 클라이언트에서 동일하게 사용하도록 하는 패턴\Composite\1
인터페이스로 인해 함께 사용하지 못하는 클래스를 함께 사용하도록 하는 패턴\Adapter\1
클래스와 객체들이 상호작용하는 방법과 역할을 분담하는 방법을 다루는 패턴\행위패턴\1
언어 규칙 클래스를 이용하는 패턴\Interpreter\1
알고리즘 골격의 구조를 정의한 패턴\Templete Method\1
객체끼리 연결 고리를 만들어 내부적으로 전달하는 패턴\Chain of Responsibility\1
요청 자체를 캡슐화해 파라미터로 넘기는 패턴\Command\1
내부 표현은 보여주지 않고 순회하는 패턴\Iterator\1
객체 간 상호작용을 캡슐화한 패턴\Mediator\0
상태 값을 미리 저장해두었다가 복구하는 패턴\Memento\0
상태가 변할 때 의존자들에게 알리고 자동으로 업데이트하는 패턴\Observer\1
객체 내부 상태에 따라서 행위를 변경하는 패턴\State\1
다양한 알고리즘을 캡슐화해 알고리즘 대체가 가능하도록 하는 패턴\Strategy\1
호스트에 객체의 내부 상태에 접근할 수 있는 방법을 제공, 호스트 객체 연산 추가\Visitor\1
기업 내 각종 어플리케이션 및 플랫폼 간으정보 전달, 연계, 통합 등 상호 연동이 가능하게 해주는 솔루션,\EAI(Enterprise Application Integration)\1
EAI 종류\p2p, Hub&Spoke, Message bus, Hybrid 등\0
